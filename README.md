# RC

RC — демонстрационный Discord-подобный проект с голосовым чатом и демонстрацией экрана.

## Возможности
- Сервер управляет клиентами и хранит данные в SQLite.
- WebRTC голос и экран через WebSocket-сигналинг.
- Клиент хранит файлы локально на компьютере пользователя.
- Windows-клиент на базе Electron.

## Структура проекта
```
RC/
├── client/
│   ├── electron/       # Desktop-оболочка для Windows
│   └── web/            # HTML/JS клиент
├── docs/
│   └── CREATION_STEPS.md
└── server/             # FastAPI + SQLite
```

## Инструкция «для чайника»

### 1. Подготовка
Установите на компьютер:
- **Python 3.11+** (https://www.python.org/downloads/)
- **Node.js 20+** (https://nodejs.org/)
- **Git** (https://git-scm.com/downloads)

Проверьте установки:
```
python --version
node --version
npm --version
```

### 2. Запуск серверной части
1. Откройте терминал и перейдите в папку проекта:
   ```
   cd RC
   ```
2. Создайте виртуальное окружение:
   ```
   python -m venv .venv
   ```
3. Активируйте окружение:
   - Windows PowerShell:
     ```
     .\.venv\Scripts\Activate.ps1
     ```
   - Windows CMD:
     ```
     .\.venv\Scripts\activate
     ```
4. Установите зависимости:
   ```
   pip install -r server/requirements.txt
   ```
5. Запустите сервер:
   ```
   uvicorn server.app.main:app --host 0.0.0.0 --port 8000
   ```

### 3. Запуск клиента (в браузере)
1. Откройте в браузере файл:
   ```
   client/web/index.html
   ```
2. Нажмите «Подключиться», затем включите микрофон/экран.

> Для полноценной работы звонков убедитесь, что сервер запущен на `localhost:8000`.

### 4. Запуск Windows-клиента (Electron)
1. Откройте терминал в папке проекта.
2. Установите зависимости клиента:
   ```
   cd client/electron
   npm install
   ```
3. Запустите десктоп-клиент:
   ```
   npm run start
   ```

### 5. Сборка установщика для Windows
В папке `client/electron`:
```
npm run dist
```
Готовый установщик появится в папке `dist/`.

## Важно про хранение файлов
Клиент хранит файлы **только локально**:
- Через кнопку «Сохранить выбранный файл» используйте системный диалог.
- Сервер хранит только метаданные клиентов и сообщений.

## Как работает голос и экран
- Используется WebRTC.
- Сервер выполняет роль сигналинга (`/ws/signaling/{room}/{client_id}`).
- Для стабильности в реальном продукте нужно добавить TURN-сервер.
